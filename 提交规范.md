# Git 提交信息规范

## 格式

```
<type>(<scope>): <subject>
<BLANK LINE>
<body>     (可选)
```

**一行不超过 72 字符。**

## Type 类型

| Type       | 含义     | 使用场景                           |
|------------|----------|-----------------------------------|
| `feat`     | 新功能   | 添加新功能、新工具、新命令           |
| `fix`      | 修复     | 修复 bug、修复行为不符合预期         |
| `refactor` | 重构     | 代码重构，不涉及功能变化             |
| `perf`     | 性能     | 性能优化                           |
| `style`    | 样式     | 代码格式、颜色、输出样式调整         |
| `docs`     | 文档     | 文档、注释、README                 |
| `chore`    | 杂务     | 构建配置、依赖管理、CI、gitignore 等 |

## Scope 范围

| Scope       | 对应项目           |
|-------------|-------------------|
| `as`        | aminos 包管理器    |
| `ls`        | ls 目录列表工具    |
| `color`     | color crate       |
| `config`    | config crate      |
| `net`       | net crate         |
| `sys`       | sys crate         |
| `source`    | aminos-source 仓库 |
| `*`         | 跨项目通用改动      |

## Subject 规范

- **中文** 描述，简洁扼要
- 不加句号
- 用祈使语气：`添加`、`修复`、`重构`、`优化`
- 不用：`添加了`、`修复了`（"了"字冗余）

## 示例

```
feat(ls): 添加 --set/--path 子命令，支持环境变量彩色显示
fix(as): 修复 as install 安装后 PATH 检测逻辑
refactor(as): 源目录拆分为 apps/ 和 tools/
style(ls): 重写 -h 帮助输出，仿 as -e 风格
docs: 补充 README 使用说明
chore: 升级 clap 到 4.6
```

## 开发中（不合并）

```
wip(as): 正在重构 install 流程
```

## 提交步骤

```bash
git add <files>
git commit -m "$(cat <<'EOF'
feat(ls): 添加 -S/--set 选项，彩色显示环境变量

配合 ls.yaml 配置文件可自定义变量配色。
EOF
)"
```
